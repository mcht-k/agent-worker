## ZADANIE
API i frontend zwracają 502 mimo że kontenery działają. Zdiagnozuj i napraw problem z nginx proxy.

## CO ZROBIĆ
1. Sprawdź czy API odpowiada bezpośrednio z pominięciem nginx:
   `curl -v http://localhost:5200/health` i `curl -v http://localhost:4200`

2. Sprawdź konfigurację nginx:
   `ls /opt/nginx/conf.d/` i `cat /opt/nginx/conf.d/vouchify-dev.conf`

3. Sprawdź czy kontenery są w sieci dev_internal:
   `docker network inspect dev_internal | grep -A 5 "vouchify\|Name"`
   Jeśli brakuje kontenerów — podłącz je:
   `docker network connect dev_internal vouchify_api_dev`
   `docker network connect dev_internal vouchify_frontend_dev`

4. Sprawdź logi nginx-proxy:
   `docker logs nginx-proxy --tail=30`

5. Jeśli problem w konfiguracji nginx — napraw plik w `/opt/nginx/conf.d/` tak żeby `proxy_pass` wskazywał na właściwą nazwę kontenera i port. Przeładuj nginx:
   `docker exec nginx-proxy nginx -s reload`

6. Zweryfikuj:
   `curl -s -o /dev/null -w "%{http_code}" https://api.dev.vouchify.mtlabs.pl/health`
   `curl -s -o /dev/null -w "%{http_code}" https://dev.vouchify.mtlabs.pl`

## CZEGO NIE RUSZAĆ
- nginx-proxy kontener (tylko reload, nie restart)
- plannify-* kontenery
- /opt/nginx/conf.d/ pliki innych projektów

## DEFINICJA UKOŃCZENIA
- [ ] `https://api.dev.vouchify.mtlabs.pl/health` zwraca 200
- [ ] `https://dev.vouchify.mtlabs.pl` zwraca 200
